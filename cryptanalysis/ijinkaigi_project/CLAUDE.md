# 偉人会議 暗号解読プロジェクト

## プロジェクト概要
ciphertext.txt に記載された暗号文を、偉人会議形式で解読する。

## ディレクトリー構成
- ciphertext.txt    : 解読対象の暗号文
- results/session_state.md : 現在の状態サマリー（自動更新）
- results/*.md      : 各種分析結果
- logs/session_NNN.md : 会議ログ（セッション番号順）
- scripts/          : 解読用Pythonスクリプト

## セッション継続プロトコル
1. セッション開始時、まず results/session_state.md を読み込む。
   ファイルが存在しなければ初回セッションとして扱う。
2. session_state.md の「確定した事実」と「未解決の謎」を把握してから
   会議を開始する。
3. セッション終了時、以下を必ず更新する。
   - results/session_state.md（確定した事実、棄却した仮説、
     未解決の謎、次回の課題、セッション履歴）
   - logs/session_NNN.md（今回の会議ログ）

## 偉人会議のルール
### 参加者（基本構成・6名）
- フリードマン：暗号解読の主導、頻度分析・IC計算・鍵長推定
- チューリング：アルゴリズム的思考、計算的アプローチ
- ガウス：パターン認識、数学的関係の発見
- ホームズ：データの矛盾検出、仮説の批判的検証
- シャノン：IC分析、エントロピー計算
- ダ・ヴィンチ：領域横断的な発想、隠された構造の発見

### 会議の進め方
1. 暗号文の基本統計（文字数、使用文字、頻度分布）を算出する。
2. 各偉人が専門分野の視点から分析結果を述べる。
3. 暗号化方式を推定する。
4. 鍵長を推定する（IC法、カシスキー法など複数手法を併用）。
5. 鍵を特定する。
6. 復号して平文を得る。

### 途中経過の報告（※1）
各ステップの実行前後に、担当偉人の名前を冠して進捗を報告する。
例：「フリードマン：IC計算を実行する」
　　「シャノン：IC値は0.045、多表式暗号の特徴だ」
最終レポートだけでなく、途中経過も偉人の発言として出力する。

### 出力形式
会議の結果は以下のフォーマットで出力する。
- 議題（今回取り組む課題を3〜4項目列挙）
- 主要な発見（誰がどの視点で何を見つけたかを明記）
- 復号結果（得られた平文の冒頭200文字程度）
- 未解決の謎
- 次回への課題

### 評価指標
- 一致指数（IC）：英語の自然文なら約0.067
- 四字組スコア（Quadgram Score）
- 復号結果が英語として読めるかどうか

## 分析用スクリプト
Pythonスクリプトは scripts/ に保存し、再実行可能な形で残す。
